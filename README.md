# Product Designer OS

> **Become a world-class Product Designer by building real products.**
>
> A project-first operating system designed to help you get a product design job as quickly as possible while building the foundations for a world-class career.

---

## Why this exists

Most product design courses teach **knowledge first**.

They look like this:

```
UX Research

↓

Wireframing

↓

UI Design

↓

Portfolio

↓

Job
```

That's not how real product teams work.

Real teams start with a problem.

They learn only what is necessary to solve it.

This repository follows the same philosophy.

```
Real Problem

↓

Build a Product

↓

Learn what the project demands

↓

Create a Case Study

↓

Build a Portfolio

↓

Get Interviews

↓

Get the Job
```

Knowledge is never the goal.

Building is.

---

# The Mission

The primary goal of this repository is simple.

> Help one motivated designer become employable at a high-quality product company in the shortest possible time.

Everything else is secondary.

---

# Who this is for

This repository is for people who:

- Know some UI but want to become Product Designers.
- Feel overwhelmed by endless resources.
- Learn best by building.
- Want to think beyond screens.
- Want to work at world-class product companies.
- Care about psychology, systems thinking, business, and technology.

This is **not** for people looking for a quick UI course or a Figma tutorial.

---

# Core Philosophy

This repository is built on five principles.

## 1. Projects pull knowledge.

Don't learn typography because it's chapter three.

Learn typography because today's project requires it.

---

## 2. Build before consuming.

For every hour spent reading, spend at least three hours building.

---

## 3. Reality over theory.

Interview users.

Ship prototypes.

Collect feedback.

Measure outcomes.

---

## 4. Evidence beats credentials.

Companies hire proof.

Not certificates.

Every project should produce something tangible.

---

## 5. Simplicity scales.

If something doesn't improve your ability to build products or get hired, remove it.
